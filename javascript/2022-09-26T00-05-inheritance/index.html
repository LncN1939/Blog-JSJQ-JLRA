Polimorfismo

Publicado el 26 de septiembre de 2022

por Jezer Mejía

¿Qué es el polimorfismo? Según wikipedia, el polimorfismo se refiere a la propiedad o interfaz que múltiples entidades pueden tener en común, de tal manera que pueden producirse distintas clases a partir de una sola. Esto es similar al principio biológico en que un organismo puede tener distintas formas o etapas, o bien puede verse como una rama biológica.

También puede definirse como la reutilización de código de una clase en otras de manera implícita; el código no se repite, sino que las propiedades y métodos son heredados de una clase a otra.

JavaScript, al igual que otros lenguajes de programación orientados a objetos, es capaz de hacer polimorfismo de clases. Para ello se hace uso de la palabra clave: extends.

Ejemplo:

Digamos que tenemos una clase Mueble con sus distintas propiedades como el nombre, material, dimensiones, etc. En principio, un mueble no debería tener ningún método per sé, pero le implementaremos uno para imprimir los detalles del mismo. 

class Mueble {
  constructor(name, material) {
    this.name = name;
    this.material = material;
  }

  imprimirDetalles() {
    console.log(`Nombre: ${this.name}\nMaterial: ${this.material}`);
  }
}

La clase Mueble sería una abstracción que nos serviría para crear consecuentes tipos de muebles, como lo puede ser una silla o una mesa. 

class Silla extends Mueble {
  constructor() {
    super("Silla", "madera");
  }
  sentar() {
    console.log(`Te has sentado en una ${this.name}`);
  }
}

En el código de arriba se crea una clase llamada Silla, que se extiende a partir de un Mueble. De esta manera, una silla tiene las mismas propiedades que un mueble: un nombre y un material; y tiene los mismos métodos: imprimir los detalles.

Además de ello, una Silla implementa un método adicional de sentar (que no es lógico ya que una silla no se sienta) que es único para la clase de Silla; un mueble no tiene este método.

A su vez, como puede observarse en el constructor, con la palabra clave super se puede acceder a la clase padre o superior, que en este caso resulta ser Mueble. Con super podemos llamar métodos, obtener las propiedades, y construir un objeto con el constructor del padre. Lo último es lo que se está haciendo en el código de arriba: crear un mueble con el nombre de "Silla" y un material de "madera".
